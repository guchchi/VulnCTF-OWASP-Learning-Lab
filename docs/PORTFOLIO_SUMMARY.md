# VulnCTF Portfolio Summary

## Project Overview

VulnCTF is a deliberately vulnerable Flask web application created for local cybersecurity learning. It includes five CTF-style challenges covering common beginner web security vulnerabilities from the OWASP Top 10.

## Why I Built It

I built this project to understand how common web vulnerabilities happen, why they are risky, and how they can be fixed securely. It demonstrates understanding of both insecure patterns and defensive coding practices.

## Vulnerabilities Covered

- SQL Injection
- Stored XSS (Cross-Site Scripting)
- IDOR (Insecure Direct Object Reference / Broken Access Control)
- Path Traversal
- Command Injection

## Cybersecurity Concepts Learned

- OWASP Top 10 awareness
- Input validation and sanitization
- Output encoding and escaping
- Authentication vs. authorization
- Server-side access control
- Secure file handling patterns
- Safe command execution practices
- Parameterized queries
- Secure coding mindset
- Threat modeling basics

## Responsible Learning Note

This project is intentionally vulnerable and should only be used locally for educational purposes. It is not intended for attacking or testing real systems without authorization.

## Portfolio Statement

This project helped me move from building web applications to understanding how they can fail when security is not designed properly. It strengthened my interest in web security, secure coding, and defensive cybersecurity.
